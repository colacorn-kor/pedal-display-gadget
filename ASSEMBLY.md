# pedal-display-gadget — 초보자용 조립 가이드

이 문서 하나만 위에서부터 차례대로 따라 하면 **디스플레이 + 버튼 + 튜너 + 비주얼라이저**가
도는 첫 프로토타입이 완성된다. 전자공작이 처음이어도 괜찮도록 "이게 뭔지 / 왜 하는지 /
어디에 꽂는지 / 제대로 됐는지 어떻게 아는지"를 단계마다 적었다.

> **핵심 안전 3가지 (이것만은 꼭)**
> 1. **MP1584(전압 변환기)는 보드에 연결하기 *전에* 출력을 정확히 5.0V로 맞춘다.** 안 맞추면
>    ESP32 보드가 즉사할 수 있다.
> 2. **첫 테스트는 USB 하나로만.** 외부 전원(5V)과 USB를 *동시에* 꽂지 않는다.
> 3. **납땜인두는 350℃ 쇳덩이다.** 받침대에 두고, 환기하고, 손 데지 않게.

> **이 가이드의 큰 그림 (데이터 흐름)**
> 기타 신호 → (TL072로 살짝 복사해서 따옴, 원래 소리는 그대로 통과) → PCM1808이 디지털로
> 바꿈 → ESP32-S3가 분석(음정/스펙트럼) → ST7796 화면에 표시. 버튼·풋스위치로 앱을 바꾼다.
> 처음엔 화면+버튼만, 그다음 전원, 마지막에 오디오 순서로 붙인다.

> **튜너 음역 = 30~1300Hz** (베이스 5현 저B 31Hz·드롭튜닝 포함). 비주얼라이저는 20Hz~20kHz.
> J201(뮤트)·SD카드·MIDI는 **나중 단계** (맨 아래 "보류 항목").

## 1. 준비물

### 1-1. 공구 (없으면 못 함)
- **납땜인두 + 납** — 모듈 핀헤더 납땜용.
- **멀티미터** — 전압 확인용. **이게 없으면 시작하지 마라.** 전압을 못 보면 보드가 죽는다.
- **브레드보드(빵판)** — 처음엔 납땜 대신 꽂아서 테스트. 큰 거 1개 권장.
- **점퍼 와이어** — 수-수(M-M), 수-암(M-F) 넉넉히. (모듈↔빵판 연결)
- **작은 일자 드라이버** — MP1584 트리머(파란 가변저항) 돌리기 + 터미널블록 조이기.
- **니퍼(소형 커터)** — 다리/와이어 자르기.
- (있으면 편함) 헬핑핸드(납땜 집게), 와이어 스트리퍼, 보안경.

## 2. 알아두면 쉬운 기초

- **GPIO** = ESP32의 "범용 입출력 핀". 번호(예: GPIO8)로 부른다. 코드가 이 핀을 켜고/끄거나
  읽는다. 보드 옆면 실크에 번호가 적혀 있다.
- **VCC / GND / 신호** = 모든 모듈은 전원(VCC, +), 접지(GND, −), 데이터선(신호) 세 종류로 연결.
  **GND는 모든 모듈이 서로 공유**해야 한다(공통 그라운드). 이게 안 되면 아무것도 안 된다.
- **풀업 + LOW 버튼** = 버튼은 "한쪽 = GPIO, 반대쪽 = GND"로 연결한다. 평소엔 내부 풀업
  저항이 핀을 HIGH(3.3V)로 유지하다가, **누르면 GND와 연결돼 LOW(0V)** 가 된다. 펌웨어가
  이 변화를 "눌림"으로 읽는다. 그래서 **외부 저항 필요 없음.**
- **멀티미터로 전압 재기** = 검정 프로브 = GND(COM), 빨강 프로브 = 재고 싶은 점. 다이얼을
  DC V(직류 전압, V⎓)에 둔다. 화면에 5.0, 3.3 같은 숫자가 뜬다.
- **브레드보드** = 가로줄(+/− 전원 레일)과 세로줄(부품 다리 묶음)로 구멍이 연결돼 있다.
  같은 세로줄에 꽂은 다리들은 서로 연결된다. 전원 레일에 5V/3.3V/GND를 깔고 거기서 따 쓴다.
- **핀헤더 납땜** = 모듈에 헤더(핀 줄)가 안 붙어 있으면 먼저 납땜해야 빵판에 꽂힌다.
- **극성 주의** = 전해 콘덴서(원통)는 +/− 있음(긴 다리 = +, 몸통에 − 줄). LED·다이오드·9V도
  방향 있음. 반대로 꽂으면 안 됨.

## 3. 모듈 식별 가이드 (어떤 핀이 뭔지)

> 모듈마다 실크(인쇄 글자) 표기가 조금씩 다르다. **항상 모듈에 적힌 글자를 기준**으로 찾고,
> 아래는 흔한 이름이다.

- **ESP32-S3-DevKitC-1**: 가장 큰 보드. USB-C 2개(보통 하나는 UART/플래시, 하나는 native USB),
  옆면에 `1, 2, 4...` GPIO 번호 실크. 전원 핀 `5V`, `3V3`, `GND` 여러 개.
- **ST7796 디스플레이**: 가장 큰 화면. 네 모듈 단자는 `GND VCC SCL SDA RST DC CS BL SDA-0`
  9개다. 여기서 **SCL = SPI 클럭(SCK), SDA = SPI 데이터입력(MOSI), BL = 백라이트,
  SDA-0 = 데이터출력(MISO/SDO, 안 씀)**. (모듈마다 실크 표기가 달라 SCK를 SCL, MOSI를
  SDA로 적는 경우가 많다. CS·DC·RST가 있으니 I2C가 아니라 4선 SPI다.)
- **PCM1808 모듈**: 작은 사각 모듈. 신호핀 `SCKI BCK LRC(LRCK) OUT(DOUT) FMT MD1 MD0`,
  전원 `VCC GND`, 아날로그 입력 `VINL VINR`(또는 L/R). `FMT/MD0/MD1` = 모드 점퍼.
- **MP1584 벅**: 아주 작은 모듈에 **파란 네모 트리머(가변저항)**. `IN+ IN− OUT+ OUT−`. 이
  트리머를 돌려 출력 전압을 맞춘다.
- **AMS1117-3.3 모듈**: 작은 3핀 레귤레이터(`IN GND OUT`). 5V→3.3V. **이번 빌드에선 안 씀**
  (PCM1808 모듈이 온보드 레귤레이터로 아날로그 3.3V를 자체 생성). 나중에 별도 클린 레일이 필요하면 사용.
- **TL072CP**: 검은 8핀 칩(DIP). 한쪽 끝에 **반원 홈/점** = 1번 핀 기준. 빵판 중앙 골에 걸쳐 꽂는다.
  - 1=OUT A, 2=IN− A, 3=IN+ A, 4=V−(GND), 5=IN+ B, 6=IN− B, 7=OUT B, 8=V+(9V)
- **ELB040202**: 배럴잭을 나사 터미널로 빼주는 어댑터. 9V 어댑터를 여기 꽂고 `+ / −` 나온다.

## 4. 안전 수칙 (다시 한 번)

- **MP1584 선세팅(5.0V) 전엔 절대 보드에 연결 금지.** (6단계에서 자세히)
- **USB와 외부 5V 동시 금지** — 백피드로 보드/PC 포트 손상 위험. 테스트는 둘 중 하나만.
- **전원 켜기 전 항상 다시 본다**: VCC↔GND 단락(쇼트) 없는지, 극성 맞는지.
- **납땜**: 인두 받침대 사용, 환기, 식기 전 만지지 않기. 납 연기 안 마시기.
- **정전기**: 작업 전 금속(수도꼭지 등) 한 번 만져 몸의 정전기 빼기.
- 멀티미터로 **부하 연결 전에** 레일 전압부터 확인하는 습관.

### 5-2. 전원 체인 (큰 그림)

```
9V 어댑터
   │
ELB040202 (배럴잭 → 터미널, 여기서 9V +/− 가 나온다)
   │
9V+ ─[ 1N5819  ▷|띠 ]─ 보호된 9V+   ← ★ 역전압 보호(직렬). 띠(캐소드)=부하(MP1584) 쪽
   │                                    (9V− = GND 는 그대로 직결)
   ├─[A]→ MP1584 IN ─→ (5.0V로 세팅) ─┬─→ DevKit 5V 핀
   │   (보호된 9V+)                    └─→ PCM1808 +5V 핀 (모듈 온보드 레귤레이터→아날로그)
   └─[B]→ 100Ω + 100µF + 100nF (RC필터) ─→ 깨끗한 9V ─→ TL072 V+(8번)

DevKit 3V3 핀 ─→ PCM1808 3.3 핀 (디지털 로직·I2S 레벨)
```

- **디지털 5V**: MP1584 → DevKit `5V` 핀. (ESP32 + 디스플레이 3.3V는 DevKit 내장 LDO가 만듦)
- **PCM1808 전원**: `+5V` ← MP1584 5V(= DevKit 5V 핀과 같은 레일), `3.3` ← DevKit `3V3` 핀, `GND` 공통.
  모듈 **온보드 레귤레이터가 +5V로 아날로그(AVDD)를 만들고**, `3.3`은 디지털/I2S 로직용.
  → **AMS1117 불필요**(모듈이 그 역할 내장, 스위처 리플도 모듈 LDO가 걸러줌).
- **op-amp 9V(깨끗)**: ELB040202의 9V를 직접 따서 RC필터 거쳐 TL072 V+. 디지털과 분리해야 조용함.
- **역전압 보호(★)**: ELB040202의 9V+ 바로 뒤에 **1N5819 쇼트키를 직렬로 1개**. 띠(캐소드)가
  부하(MP1584) 쪽을 향한다. 어댑터를 거꾸로 꽂아도 차단돼 보드를 지킨다. 순방향 강하 ~0.4V라
  버크 입력이 ~8.6V로 들어가지만 5V 만드는 덴 지장 없고, 이 한 개가 버크·op-amp 9V 탭까지 전체를 보호한다.
- **공통 그라운드(스타 그라운드)**: 모든 GND(어댑터·MP1584·AMS1117·DevKit·PCM1808·TL072·잭)를
  **한 점에서** 묶는다. 가능하면 전원 입구 근처. ★ 이거 빼먹으면 잡음·오작동의 90%가 여기서 나온다.

## 6. 조립 순서 (단계마다 검증하고 다음으로)

> 각 단계는 **목표 → 배선 → ✅확인(이렇게 보여야 정상) → 흔한 실수** 형식이다. ✅가 안 되면
> 다음으로 넘어가지 말고 그 단계에서 해결.

### Step 0 — 사전 준비
**목표:** 모듈에 헤더 납땜, 전압 선세팅, 모드 점퍼 확인. (여기서 시간 들여두면 뒤가 편하다.)

1. **핀헤더 납땜**: 헤더 안 붙은 모듈(보통 MP1584·PCM1808 등)에 헤더를 납땜한다.
   빵판에 헤더를 꽂아 고정한 뒤 모듈을 얹고 각 핀을 한 번씩 지져주면 수평이 잘 맞는다.
2. **MP1584를 5.0V로 선세팅** (아래 ⚠ 절차 — 가장 중요):
   - ① MP1584 IN+/IN−에만 9V를 넣는다. **OUT은 어디에도 연결하지 않는다.**
   - ② 멀티미터를 OUT+(빨강)·OUT−(검정)에 대고, 작은 드라이버로 파란 트리머를 천천히 돌린다.
   - ③ 화면이 **정확히 5.0V**가 될 때까지 맞춘다. (4.9~5.1V면 OK, 5.2V 넘기면 위험)
   - ④ 9V를 빼고, **그다음에** 5단계 배선에서 DevKit에 연결한다.
   - ⚠ 이 순서를 안 지키면 트리머가 우연히 고전압으로 맞춰진 상태에서 보드에 5V 초과가 들어가 ESP32가 죽는다.
3. **PCM1808 모드 점퍼**: **ESP32가 I2S 마스터, PCM1808은 슬레이브**가 되도록 둔다(즉, 클럭은
   ESP32가 만들고 PCM1808은 받는다). `FMT = I2S`. 모듈마다 `MD0/MD1` 점퍼 위치가 다르니 모듈
   실크/데이터시트의 "Slave / I2S 24-bit" 설정을 따른다. (모듈이 마스터로 하드와이어돼 있으면
   클럭 충돌 → 반드시 슬레이브로)

✅ **확인:** MP1584 OUT이 5.0V. PCM1808 점퍼 = 슬레이브 + I2S.
🚫 **흔한 실수:** 트리머를 끝까지 돌려 12V 가까이 만든 채 연결 / 점퍼를 마스터로 둠.

### Step 1 — 디스플레이 첫 불 (USB만)
**목표:** 화면이 켜지고 글자가 뜨는 것 확인. **외부 전원 없이 USB만으로.** 전원 체인은 아직 안 만든다.

**배선 (DevKit ↔ ST7796, 점퍼 8개):** 네 모듈 단자 순서 그대로 정리했다.

| 모듈 단자 | 무슨 신호 | → DevKit |
|-----------|-----------|----------|
| GND   | 접지 | GND |
| VCC   | 전원 | 3V3 |
| SCL   | SPI 클럭(SCK) | GPIO12 |
| SDA   | SPI 데이터입력(MOSI) | GPIO13 |
| RST   | 리셋 | GPIO14 |
| DC    | 데이터/커맨드 | GPIO21 |
| CS    | 칩 셀렉트 | GPIO2 |
| BL    | 백라이트 | GPIO1 |
| SDA-0 | 데이터출력(MISO/SDO) | **연결 안 함** |

> `SDA-0`(=MISO)는 화면에 쓰기만 하므로 비워둔다(펌웨어 `miso=-1`). 터치 단자가 따로 있으면
> 그것도 지금은 안 씀. VCC는 모듈 사양 확인 — 대부분 3.3~5V 허용. 우선 DevKit `3V3`에 연결
> (로직과 같은 전압이라 안전). 너무 어두우면 뒤에서 5V로 바꿔도 됨.

**펌웨어 올리기:**
- ESP-IDF 설치 후 (`README.md` 참고), 저장소에서:
  `idf.py set-target esp32s3` → `idf.py build` → `idf.py -p COM포트 flash monitor`
  (윈도우는 `COM5` 같은 식. 안 잡히면 BOOT 버튼 누른 채 USB 연결 후 플래시)
- (선택) 더 단순한 점등만 보고 싶으면 `display_bringup.c` 맨 아래 주석의 임시 `app_main`
  (HELLO ST7796)을 쓴다. 굳이 안 해도 **Step 2에서 런처가 뜨면 디스플레이는 정상**이다.

✅ **확인:** 백라이트 켜짐 + 화면에 글자("HELLO ST7796" 또는 Step 2의 "PEDAL DISPLAY") 표시.
🚫 **흔한 실수:**
- 화면 깜깜 → VCC/GND/백라이트(GPIO8) 배선, 케이블 접촉 확인.
- 색이 반전/빨강↔파랑 → 코드의 `esp_lcd_panel_invert_color` / `rgb_ele_order`(BGR↔RGB) 조정.
- 점퍼 한 칸 밀려 꽂힘(가장 흔함) → SCK/MOSI/CS/DC/RST 다시 한 칸씩 대조.

### Step 2 — 전체 펌웨어 + 버튼 테스트 (USB)
**목표:** 정식 펌웨어로 런처·버튼·풋스위치·홈이 동작하는지.

**추가 배선 (버튼 7개):** 각 택트/풋스위치의 **한 다리 = 해당 GPIO, 다른 다리 = GND**.
- 상=4, 하=5, 좌=6, 우=7, 확인=16, 홈=15, 풋스위치=17.

**조작법(이게 ②단계 입력 모델):**
- 상/하(또는 좌/우)로 메뉴 커서 이동, **확인(OK)으로 앱 진입**.
- 방향키를 **꾹 누르면 오토리피트**(빠르게 반복 이동).
- **풋스위치 숏 = 다음 앱 순환 / 롱(0.5초) = 튜너로 점프.**
- **홈 숏 = 공통 팝업**(맨 위 Exit, 확인 누르면 홈으로) / **홈 롱 = 즉시 홈 복귀**(먹통 탈출용).

✅ **확인:** "PEDAL DISPLAY" 런처가 뜨고, 버튼으로 커서가 움직이고, 확인으로 앱에 들어가고,
홈 숏으로 팝업이 뜬다. (오디오는 아직 안 붙였으니 튜너·모니터는 "—"/빈 화면이 정상.)
🚫 **흔한 실수:**
- 특정 버튼 무반응 → 그 GPIO↔GND 배선 확인(버튼 두 다리 중 *대각선* 다리가 한 쌍).
- 가만히 있는데 입력이 들어옴 → 그 핀이 GND/다른 핀에 닿았는지(쇼트) 확인.
- 참고: 뮤트(GPIO21)는 아직 미배선이라 **튜너 점프 시 소리 변화는 없다(정상)** — GPIO만 토글됨.

### Step 3 — 전원 레일 만들기
**목표:** USB 없이 9V 어댑터만으로 보드가 돌게.

**배선:**
1. 9V 어댑터 → ELB040202 → 9V(+/−) 확보.
2. **역전압 보호 다이오드(1N5819)**: 9V+ 경로에 **직렬**로 끼운다 — 띠(캐소드)가 MP1584 쪽.
   빵판이면 +레일을 끊고 그 사이에 걸쳐 꽂아 `9V+ → 다이오드 → 보호된 +레일`로 흐르게.
3. 보호된 9V+ → MP1584 IN+, 9V− → MP1584 IN−. (MP1584는 Step 0에서 5.0V 세팅 완료)
4. MP1584 OUT(5V) → DevKit `5V` 핀. (PCM1808 `+5V`는 4단계에서 이 5V 레일에 연결)
5. **모든 GND를 한 점에 공통 결선** (스타 그라운드).

✅ **확인 (부하 연결 *전에* 멀티미터로):**
- 다이오드 뒤(MP1584 IN+) = **~8.6V**(정방향 정상). 0V거나 9V가 안 나오면 다이오드 방향(띠) 반대.
- MP1584 OUT = **5.0V**. 부팅 후 DevKit `3V3` 핀 = **3.3V**.
- 그다음 USB 빼고 9V만으로 DevKit이 부팅돼 런처가 뜨는지.
🚫 **흔한 실수:**
- **USB와 9V 동시 연결**(백피드) — 한 번에 하나만.
- 5V 핀에 5V 초과가 들어감 → Step 0 선세팅 누락. (보드 사망 신호)
- GND 따로 놀음 → 부팅 안 되거나 잡음 폭발.

### Step 4 — PCM1808 (오디오 디지털 변환) 붙이기
**목표:** I2S로 ESP32가 오디오 데이터를 받기 시작.

**배선 (I2S 4선 + 전원):**

| ESP32 GPIO | 신호 | → PCM1808 |
|------------|------|-----------|
| 8 (MCLK)  | 시스템 클럭 | SCKI |
| 9 (BCK)   | 비트 클럭   | BCK |
| 18 (WS)   | 워드 클럭   | LRC(LRCK) |
| 10 (DIN)  | 데이터(ESP가 읽음) | OUT(DOUT) |

- 전원: PCM1808 `+5V` = MP1584 5V 레일(= DevKit 5V 핀), `3.3` = DevKit `3V3` 핀, `GND` 공통.
  모듈 온보드 레귤레이터가 `+5V`로 아날로그를 만들고 `3.3`은 디지털/I2S 로직. **AMS1117 안 씀.**
  (⚠ 모듈 `+5V`핀과 `3.3`핀이 내부에서 쇼트 아닌지 저항 모드로 한 번 확인 — 분리돼 있어야 둘 다 급전)

✅ **확인:** 입력에 아무것도 안 꽂은 채 **Sound Monitor** 앱 진입 → 바닥 잡음/평탄한 스펙트럼이
보이면 I2S 정상. (시리얼 모니터에 I2S overflow 로그가 *계속* 뜨면 클럭/슬롯/점퍼 재확인)
🚫 **흔한 실수:** SCKI에 MCLK(GPIO16) 안 줌 / 점퍼가 마스터 / DOUT↔DIN 방향 헷갈림.

---

### Step 5 — 아날로그 프론트엔드 + 실제 기타 신호
**목표:** 기타를 꽂아 튜너·스펙트럼이 반응. **원래 소리는 손상 없이 통과**시키면서 신호만 살짝 따온다.

PCM1808 입력 사양 = **single-ended 3Vp-p 풀스케일**. TL072로 기타 신호를 버퍼+게인해서 넣는다.

**① 가상 그라운드 4.5V 만들기 (TL072 B쪽 = 버퍼):**
```
9V ─ 10k ─┬─ 10k ─ GND          ← 분압으로 4.5V
          ├─ 100µF ─ GND         ← 큰 흔들림 잡기 (전해, 극성 주의)
          ├─ 100nF ─ GND         ← 고주파 잡기 (세라믹)
          └─ TL072 +inB(5번)
TL072 outB(7번) ─┬─ -inB(6번)     ← 출력을 −입력에 직결 = 전압 따라가기(버퍼)
                 └─ ★ 이 점이 "깨끗한 4.5V(Vref)". 이후 4.5V 기준은 전부 여기서 딴다.
```

**② 신호 버퍼 + 게인 (TL072 A쪽):**
```
입력잭 Tip ──┬───────────────────────────→ 출력잭 Tip   (패스스루: 톤 무손상 직결)
            │                                    └─(나중) J201 뮤트 션트
            └─ 1µF ─┬─ TL072 +inA(3번)
                    └─ 1MΩ ─ Vref(4.5V)          ← +입력 바이어스 & 입력임피던스 ~1MΩ
TL072 -inA(2번) ─┬─ 15k(Rf) ─ outA(1번)          ← 피드백
                └─ 10k(Rg) ─ Vref(4.5V)          ← ★ GND가 아니라 4.5V에 연결!
TL072 outA(1번) ─ 1µF ─→ PCM1808 VINL            게인 = 1 + 15k/10k = 2.5×
TL072 V+(8번) = 깨끗한 9V,  V−(4번) = GND
```

> ★ **단일전원 op-amp 핵심 (꼭 읽기):** 9V 한 종류만 쓰는 회로라, 게인 저항 Rg(10k)를
> **GND가 아니라 4.5V(Vref)에 연결**해야 한다. 그래야 신호가 없을 때 출력이 4.5V 한가운데
> 머물고, 신호가 들어오면 ±로 흔들린다. (Rg를 GND로 잡으면 출력이 4.5V×2.5=11.25V로
> 9V 천장에 붙어 찌그러진다.) 이게 단일전원 비반전 앰프의 표준 처리다.
> *대안:* Rg를 GND로 두되 그 사이에 10µF를 직렬로 넣어도(직류 차단) 같은 효과.

> **⚠ 저역(30Hz) 보존:** 튜너가 베이스 저B(31Hz)·드롭튜닝까지 잡아야 하므로 입력 커플링
> 캡(1µF)을 **더 작은 값으로 줄이지 말 것.** 1µF + 1MΩ = 코너 0.16Hz라 30Hz를 안 깎는다.
> 출력 쪽 1µF도 PCM1808 입력 임피던스와 합쳐 ~15Hz 이하 코너인지만 확인(모듈 바이어스망에
> 따라 다름). 안티앨리어싱은 PCM1808 내장 필터가 처리 → 외부 필터 불필요.

✅ **확인:** 기타 연결 →
- **Tuner** 화면에서 음정 검출(센트 니들이 반응).
- **Sound Monitor**에서 스펙트럼이 소리에 반응.
- **저역 확장**: 베이스 저E(~41Hz)·드롭D(~73Hz), 가능하면 5현 저B(~31Hz)까지 음 이름이
  *안정적으로* 뜨는지. 저역에서 음이 흔들리거나 한 옥타브 위로 튀면 → 커플링 캡/게인/노이즈 점검.
🚫 **흔한 실수:**
- 출력이 항상 한쪽으로 쏠려 반응 안 함 → ★ Rg를 4.5V에 안 걸고 GND에 걸었음.
- 과반응/파형 깨짐(클리핑) → 게인 ↓ (Rf를 10k로 낮춰 ×2로).
- Vref가 4.5V가 아님 → 분압 저항(10k+10k)·버퍼(B쪽) 배선 확인.
- 참고: 튜너 갱신 주기 24ms(약 42Hz). 저역은 원래 천천히 맞춰지므로 체감 무방.

## 7. 보류 항목 (지금 조립 안 함)

- **J201 뮤트 (GPIO3)**: J201은 도착함. 다만 공핍형이라 **게이트를 음전압으로 꺼야** 하는데
  ESP32 GPIO는 0~3.3V(양수)만 낸다. 그래서 G3는 "제어 입력"일 뿐이고, 게이트에 음전압을
  만드는 드라이브(+RC 소프트램프) 회로를 **별도 설계**해야 실제 뮤트가 된다. 그 회로 확정 전엔
  뮤트 미동작(튜너·비주얼라이저·앱 전환엔 지장 없음).
- **SD 카드 (핀 배정됨, 펌웨어는 스텁)**: 어댑터 6핀 = `SCK=G12 · MOSI=G13`(디스플레이와 SPI
  버스 공유) + **SD 전용 `MISO=G11 · CS=G47`** + `VCC · GND`. VCC는 모듈에 레귤레이터/레벨
  시프터가 있으면 5V, 맨 어댑터면 3.3V. 배선은 지금 해둬도 되지만 펌웨어 SD 마운트
  (`esp_vfs_fat_sdspi_mount`)는 아직 미구현 스텁이라 코드 붙기 전까진 이미지 화면이
  플레이스홀더. (구현 시 디스플레이 SPI 버스 config에 `miso=G11` 추가 필요.)
- **MIDI (Phase 2)**: 파서·매핑은 완료, UART RX + 옵토 절연 회로 미연결. 옵토커플러
  (6N138/H11L1)·TRS-MIDI 잭은 Phase 2 부품.

## 9. 용어 사전 (빠른 참고)

- **GPIO**: ESP32의 입출력 핀(번호로 부름).
- **VCC / GND**: 전원(+) / 접지(−). GND는 전 모듈 공유.
- **풀업**: 평소 HIGH로 당겨두는 저항. 버튼 누르면 GND로 LOW. (여기선 내부 풀업 사용)
- **I2S**: 디지털 오디오 전송 규격. MCLK/BCK/LRCK(클럭들) + 데이터선.
- **마스터/슬레이브**: 클럭을 *만드는* 쪽 = 마스터(여기선 ESP32), *받는* 쪽 = 슬레이브(PCM1808).
- **LDO / 벅(buck)**: 둘 다 전압 강하기. LDO(AMS1117)는 조용하지만 열, 벅(MP1584)은 효율 좋고 시원.
- **op-amp**: 신호 증폭/버퍼 칩(TL072). 통과 톤 안 건드리고 신호만 따오는 데 씀.
- **가상 그라운드(Vref 4.5V)**: 단일전원에서 신호가 ±로 흔들릴 "기준 중심점".
- **AC 커플링**: 콘덴서로 직류(DC)는 막고 신호(AC)만 통과. 캡 값이 작으면 저역이 깎임.
- **스타 그라운드**: 모든 GND를 한 점에서 묶기. 잡음·오작동 예방의 핵심.
- **백피드**: 두 전원을 동시에 물려 한쪽이 다른 쪽으로 역류 → 손상 위험.
- **디바운스**: 버튼 접점 떨림을 무시하는 짧은 지연(여기선 30ms).
