## 1. INST/LINE 게인 스위치 ★지금 추가

### 원리
현재 게인 = 1 + R15k/R10k_rg = **2.5× 고정**. 이걸 2단 전환한다.
- **INST** (기타 직결, 앞단): 신호 약함(~100mV) → **높은 게인** 필요
- **LINE** (이펙터 뒤단, 다수): 신호 셈(수백mV~1V) → **낮은 게인** (안 그러면 ADC 클리핑)

### 회로 변경 (R10k_rg를 스위치로 교체)
기존 `R10k_rg`(AIN_N↔VREF, 10k 고정)를 **SPDT 스위치 + 저항 2개**로 바꾼다:

```
SW_GAIN (SPDT):
  공통(pole)  → VREF (4.5V)
  throw LINE  → R_line → AIN_N   (큰 저항 = 낮은 게인)
  throw INST  → R_inst → AIN_N   (작은 저항 = 높은 게인)

게인 = 1 + R15k / (선택된 Rg)
```

### 예시 값 (실측 후 튜닝 전제)
| 모드 | Rg | 게인 | 비고 |
|---|---|---|---|
| LINE | R_line ≈ 15k | ~2.0× | 핫한 신호, 클리핑 방지 |
| INST | R_inst ≈ 2.2k | ~7.8× | 약한 기타 신호 부스트 |

> 값은 시작점. J1에 실제 기타/이펙터 물리고 시각화 클리핑 여부 보며 조정.
> R15k(피드백)·나머지 신호경로는 **그대로**. 바뀌는 건 Rg 자리뿐.

### 검토 규칙 유지
- 선택된 Rg의 반대끝은 **반드시 VREF(4.5V)**, GND 아님 (단일전원 포화 방지).
- 스위치 접점 불량 대비: 브레드보드 점퍼 짧게.

## 2. SD 모듈 배선 (SSOT §6 그대로, 아직이면)
```
J3.SCK  → LCD_SCLK 네트(G12, U2.SCL과 공유)
J3.MOSI → LCD_MOSI 네트(G13, U2.SDA와 공유)
J3.MISO → U1.G11   (SD 전용)
J3.CS   → U1.G47   (SD 전용)
J3.VCC  → +3V3     (모듈이 3.3 로직이면. 5V+온보드LDO 모듈이면 +5V)
J3.GND  → GND
```
> 펌웨어 스텁이라 지금 기능은 없음. 물리 배선만. VCC는 모듈 사양 확인(미결 항목).

## 3. 코덱 자리 확보 (핀 비우기 — 실제 칩은 Phase 2)
나중에 코덱(예: 출력용 PCM5102 DAC, 또는 입출력 ES8388) 넣을 때 갈아엎지 않으려면
**아래 핀을 지금부터 다른 용도로 쓰지 말 것**:

- **I2S 데이터 추가 1핀** — 코덱 DAC는 ESP→코덱 방향 데이터선(DOUT)이 필요. 현재 I2S(MCLK G8/BCK G9/WS G18)는 **공유 가능**, DIN(G10)은 PCM1808 전용이므로 **코덱용 별도 데이터 핀 1개 예약**. 권장 예약: **G40**.
- **I2C 제어 2핀** — ES8388류는 레지스터 설정에 I2C 필요. 권장 예약: **G41(SDA), G42(SCL)**. (PCM5102 쓰면 I2C 불필요)
- **보드/브레드보드 공간** — 코덱 칩 + 헤드폰 앰프 + 커플링 캡 자리. AUX/헤드폰 3.5mm 잭 2개 물리 공간.
- **전원** — 코덱 3.3V(일부 1.8V) 여유. 현재 3V3 레일에 코덱 몫 감안.

> 브레드보드에선 "이 핀 줄들을 비워둔다"가 곧 자리 확보. G40/41/42 근처를 남겨두면 됨.

## 4. 본체 vs 외부 (최종 물리 구성 정리)
- **본체 온보드**: 풋스위치(SW7, G17) + **INST/LINE 게인 스위치**(신규). 이 둘만.
- **외부(Phase 2)**: 6키 = TRS 저항 래더 컨트롤러. AUX/헤드폰 3.5mm(코덱 후 동작).
- **프로토타입 현재**: 6버튼(SW1~6)은 **당분간 온보드 유지**(펌웨어 동작). TRS 전환은 펌웨어 ADC 리워크 후.

## 5. 6키 TRS 저항 래더 — ★지금 하드웨어 배선 (케이블만 미연결)
> 컨트롤러보드 + 메인 TRS잭은 지금 배선. **메인↔컨트롤러 TRS 케이블만 미연결** 상태로 둔다.
> 케이블 연결 = 펌웨어 ADC 리워크 + UP버튼 제거가 전제. 그 전까진 6버튼(SW1~6) 그대로 동작.

### TRS 3선 용도
```
Tip    = 신호 (래더 출력 → 메인 ADC 입력)
Ring   = +3V3 (메인 → 컨트롤러, 래더 전원)
Sleeve = GND
```

### 메인 브레드보드 — TRS 잭
```
TRS_jack.Sleeve → GND
TRS_jack.Ring   → +3V3
TRS_jack.Tip    → U1.G4   ★ADC1_CH3. 지금은 UP버튼과 공유(케이블 미연결이라 무해)
```
> ⚠ 케이블 연결 시 래더가 G4를 구동 → UP버튼과 충돌. 활성화 단계에서 UP버튼 제거 + 펌웨어 ADC 전환.
> ADC1 전 채널 사용 중이라 신규 전용핀 없음 → 대체될 버튼핀(G4) 재사용이 최선.

### 컨트롤러 브레드보드 — 6키 저항 래더
```
래더 전원: 케이블 Ring(+3V3) —[Rtop ~3.3k]—+—→ 케이블 Tip(신호)
                                          |
   6버튼 각각 Tip↔Sleeve(GND) 사이 서로 다른 직렬저항으로 분압:
   예) 0Ω, 1k, 2.2k, 4.7k, 10k, 22k  → 버튼별 고유 전압 6단
(동시입력 불가 — 저항 래더 특성. 확인/방향 단일 입력용)
```
- 버튼↔ui_event 매핑(UP/DOWN/LEFT/RIGHT/OK/HOME)은 펌웨어 ADC 판정 로직에서 지정.
- 저항값은 시작점 — 실측 전압 구간이 충분히 벌어지는지 보고 조정.

### 펌웨어 (나중, 별도 Codex 지시서)
- `input_task`에 ADC 원샷 읽기 경로 추가 → 전압 구간 판정 → 기존 `EV_*` 발행.
- 디바운스/오토리피트 로직은 기존 것 재사용. 6개 GPIO 버튼 경로는 제거 or 공존 선택.
